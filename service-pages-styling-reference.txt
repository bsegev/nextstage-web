3D-CARD MOBILE OPTIMIZATION:
- CardContainer: className="inter-var py-0 w-full h-[420px]" (standard)
- CardContainer: className="inter-var py-0 w-full h-[480px]" (for longer content)
- CardContainer: containerClassName="py-0 flex items-start justify-start w-full h-full"
- CardBody: Override default h-96 w-96 with w-full h-full min-w-[320px] max-w-none
- Mobile Safety: min-w-[320px] prevents card cutting on small screens
- Height Control: Fixed heights ensure consistent card heights (adjust based on content length)
- Content Length: Longer descriptions require taller cards (480px vs 420px)
- Padding Override: py-0 removes default CardContainer padding that causes mobile issues

CARD HEIGHT GUIDELINES:
- Standard Content (≤50 chars description): h-[420px]
- Long Content (>80 chars description): h-[480px]
- Extra Long Content (>120 chars): h-[520px] or higher
- Test on mobile to ensure no content cutting

Card Internal Spacing:
- Container: p-6 lg:p-8
- Title Margin: mb-3
- Context Margin: mb-4
- Visual Element: mb-4
- Description Margin: mb-5
- Button Gap: gap-3

VISUAL EFFECTS & ANIMATIONS
============================

BACKGROUND EFFECTS
-------------------
Hero Background Layers:
1. Background Image: bg-cover bg-center bg-no-repeat
2. Base Overlay: bg-background/70 sm:bg-background/60
3. Gradient Overlay: bg-gradient-to-br from-background/90 via-background/70 to-accent/30

Floating Orbs:
- Primary: w-48 h-48 sm:w-72 sm:h-72 bg-gradient-to-br from-accent/20 to-accent/10
- Secondary: w-64 h-64 sm:w-96 sm:h-96 bg-gradient-to-tr from-accent/15 to-accent/8
- Animation: animate-pulse with custom durations (4s, 6s)

Strategic Flow Lines (SVG):
- Gradient: linearGradient from transparent to accent with opacity variations
- Animation: pathLength and opacity animations with staggered delays
- Timing: 8+ second durations with infinite repeat

Grid Pattern:
- Pattern: linear-gradient overlays for subtle texture
- Size: bg-[size:4rem_4rem] sm:bg-[size:6rem_6rem]
- Opacity: Very low (0.015) for subtle effect

Metrics Background Effects:
- Data Streams: Animated horizontal lines with gradient opacity
- Fast Streams: 8 lines with 3-4s duration, linear movement
- Slow Streams: 6 lines with 8-9s duration, wider movement
- Floating Particles: Fixed positioned dots with vertical float animation
- Connection Lines: Horizontal lines between desktop metrics with data pulse
- Card Backgrounds: Multi-layer gradients with backdrop-blur-xl

ANIMATION PATTERNS
------------------
Mount Animations:
- Pattern: opacity-0 translate-y-8 → opacity-100 translate-y-0
- Duration: transition-all duration-1000
- Staggered Delays: 0ms, 300ms, 500ms for sequential elements

Card Hover Effects:
- Scale: hover:scale-[1.02]
- Shadow: hover:shadow-2xl hover:shadow-obsidian/[0.06]
- Transform: hover:-translate-y-2
- Duration: transition-all duration-1000 ease-out

Button Animations:
- Hover Transform: group-hover:-translate-y-1
- Active Scale: group-active:scale-95
- Glow Effects: opacity-0 group-hover:opacity-100
- Icon Movements: Subtle translations and rotations

Metrics Animations:
- Counter Animation: Eased number counting with staggered delays
- Card Hover: Scale animation (1.05) on metric values
- Glow Effects: Pulsing box-shadow on hover with 2s duration
- Bottom Accent: Animated gradient line on card hover
- Connection Pulse: Moving dot along connection lines (4s duration)

RESPONSIVE BREAKPOINTS
======================

BREAKPOINT STRATEGY
-------------------
- Mobile First: Base styles for mobile (320px+)
- sm: 640px+ (Small tablets, large phones)
- md: 768px+ (Tablets)
- lg: 1024px+ (Small desktops)
- xl: 1280px+ (Large desktops)
- 2xl: 1536px+ (Extra large screens)

TEXT SCALING PATTERNS
---------------------
Mobile Enhanced Pattern:
- Mobile: Larger base size for readability
- Desktop: Progressive scaling with breakpoints
- Example: text-4xl sm:text-3xl md:text-4xl lg:text-5xl xl:text-6xl

Consistent Pattern:
- Same size across breakpoints with occasional large screen boost
- Example: text-lg sm:text-lg md:text-xl lg:text-2xl

Progressive Pattern:
- Steady increase across breakpoints
- Example: text-sm sm:text-base md:text-lg lg:text-xl

LAYOUT RESPONSIVE PATTERNS
---------------------------
Container Widths:
- Mobile: Full width with padding
- Desktop: Max-width constraints with auto margins
- Hero: Asymmetric positioning (left-aligned on desktop)

Grid Patterns:
- Mobile: Single column (grid-cols-1)
- Tablet: Two columns (md:grid-cols-2)
- Desktop: Four columns (lg:grid-cols-4)

Metrics Grid Patterns:
- Mobile: Single column (grid-cols-1) with gap-4
- Desktop: Three columns (grid-cols-3) with gap-6
- Layout Switch: lg:hidden for mobile, hidden lg:block for desktop

Spacing Adjustments:
- Mobile: Tighter spacing for screen real estate
- Desktop: More generous spacing for visual breathing room
- Pattern: py-20 sm:py-24 md:py-32 lg:py-40

ACCESSIBILITY STANDARDS
========================

TOUCH TARGETS
-------------
- Minimum Height: min-h-[56px] for all interactive elements
- Touch Manipulation: touch-manipulation class for better mobile performance
- Focus States: focus-visible:ring-2 with appropriate colors

TEXT CONTRAST
-------------
- Primary Text: High contrast ratios (obsidian on light backgrounds)
- Secondary Text: Reduced opacity but still readable (70-80%)
- Interactive Text: Clear hover states with color/opacity changes

KEYBOARD NAVIGATION
-------------------
- Focus Indicators: Visible focus rings on all interactive elements
- Focus Management: Proper tab order and focus trapping where needed
- Outline Removal: focus:outline-none only when focus-visible is provided

SCREEN READER SUPPORT
---------------------
- Semantic HTML: Proper heading hierarchy (h1, h2, h3)
- Alt Text: Descriptive alt attributes for images
- ARIA Labels: Where semantic HTML isn't sufficient
